BYAI Design System v1

"One system. Every interface. BYAI, Telos AI, LeadSpark."

一套规范，统驭百应、Telos AI、LeadSpark 的所有界面。

B2B SaaS / AI-native Agent 产品的视觉与交互规范，对标 Linear / Vercel / Stripe / Anthropic 的内部 Design Infrastructure 标准。 定位：给 AI 直接消费的 design spec。任何 Agent 读完本包后， 都能输出视觉、交互、文案层面一致的界面。

设计锚点

  • 视觉气质：Anthropic editorial warmth + Linear information density
  • 色彩模式：浅色专用（暗色模式非本期交付目标）
  • 语言：简体中文 + English（双字体堆栈）
  • 产品形态：Hybrid —— Dashboard 主视图 + 常驻 Agent Copilot 侧栏
  • 密度档位：Compact / Comfortable (default) / Spacious 三档
  • 色觉：amber / blue 为主二元轴（creator 红绿色弱，拒绝 red/green 单编码）

使用本包的三种方式

方式 A · AI 全量吃入（最可靠）

把 README + spec/ 全部内容拼接为 system prompt 喂给 AI。 Token 量约 60–80K，适合 Claude / GPT-4o 级模型。

方式 B · 按需加载（推荐，省 token）

第一轮只喂 README + spec/01-philosophy.md + tokens/tokens.json + spec/appendix-c-ai-consumption.md。AI 遇到具体任务时，按下表按需拉取单模块：

任务类型按需加载文件
画 Button / Input / Table 等具象组件06-components.md
画 Streaming / Tool call / Citation 等 Agent 界面07-agent-patterns.md
处理 Loading / Empty / Error 态08-states.md
选图表类型、画仪表板09-data-viz.md
写按钮文案、错误提示10-voice-microcopy.md

方式 C · 视觉 Ground Truth（最稳）

让 AI 先读 pages/ 下的 12 张基线 HTML，再读 spec。AI 会以像素级范本为参照生成新页面， 一致性最高。

人类用户直接双击根目录 index.html 即可进入所有页面的入口；双击 packages/ui/showcase.html 进入 @byai/ui 组件画廊。

核心约定（读这份就够上手）

1 · Token 命名规则

三层架构，严格单向依赖：

Primitive (原子)          Semantic (语义)             Component (组件)
color.warm-grey.100  →   bg.canvas                →  (直接使用语义层)
color.warm-grey.900  →   fg.default               →
color.amber.600      →   intent.positive          →
color.blue.700       →   intent.info              →

规则：UI 代码里禁止直接引用 primitive，必须通过 semantic token 访问。 只有在定义 semantic 时才能引用 primitive。这条是 AI 最容易违反的，务必校验。

2 · 色觉差异友好

本系统用 amber / orange ↔ blue / cyan 作为二元对比轴， 不使用 red / green 作为唯一状态区分。 Danger 仍使用红色但必须伴随 图标 + 文字 双编码，绝不孤立用色。

3 · 字体堆栈

用途堆栈
English display (editorial)Fraunces, Instrument Serif, Georgia, serif
English body / UIGeist, Söhne, ui-sans-serif, system-ui, sans-serif
English monoJetBrains Mono, Geist Mono, ui-monospace, monospace
Chinese allMicrosoft YaHei, 微软雅黑, PingFang SC, Source Han Sans CN, sans-serif

4 · 四条不可违反的硬规

1间距只能取 space.* token 值（4 的倍数）

违反后果：视觉节奏崩坏。

2颜色只能用 semantic token，禁止 hex 字面值

违反后果：语义漂移、品牌微调全量返工。

3Radius 只能取 token 定义的档位（不能自由 5px）

违反后果：圆角节奏不统一。

4组件 state 必须覆盖 5 态

default / hover / active / focus-visible / disabled。缺一态即键盘用户受损。
