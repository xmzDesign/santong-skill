# 01 · Philosophy & Principles
> 产品哲学与五条 Design Tenet

本规范面向一类具体产品：**B2B SaaS / AI-native Agent 平台**。这类产品有三个明显特征——

1. **用户是专业人士，每天在界面里泡 6 小时以上** → 信息密度优先于"留白美学"
2. **Agent 行为不可预知** → UI 必须让用户随时理解"AI 在做什么、做到哪步、我能否干预"
3. **决策链条长、数据高价值** → 信任感是第一设计目标，花哨 > 严肃 永远错

---

## 五条 Design Tenet（按优先级降序）

当两条原则冲突时，编号小的赢。AI 遇到无法决断的设计取舍，查这里。

### Tenet 1 — Clarity over Cleverness（清晰 > 聪明）

> "如果一个专业用户需要第二次看，才能理解界面在说什么，我们就输了。"

**具体表现**：
- 标签用**名词短语**，不用隐喻（用「导出 CSV」不用「打包带走」）
- 数字旁必须有单位（`$12,450` `3.2 s` `1.2 GB` ——永远不出现裸数字）
- 状态颜色必须伴随图标（色觉差异用户能识别）
- 图表默认显示数值，不强迫 hover 才能看清

**反例**：Dashboard 上一个大数字 `2,341`，没有标签、没有单位、没有对比。

---

### Tenet 2 — Density as Respect（密度即尊重）

> "专业工具的用户讨厌滚动、讨厌翻页、讨厌"加载更多"。他们想一屏之内看到所有关键信息。"

**具体表现**：
- 默认密度是 **Comfortable**，不是 Spacious
- 表格单行高度 **36px**，不是 48px 或 56px
- 列表行内直接显示关键元数据，不依赖 tooltip
- Modal 默认尺寸比业界均值小一档，因为内容该紧凑
- **禁止**无意义的 Hero 大 banner 和装饰性占位插画

**反例**：一个销售漏斗页面，首屏 70% 是标题 + 装饰图，数据只占 30%。

---

### Tenet 3 — Agent Transparency（Agent 必须可解释）

> "用户对 AI 最大的恐惧是'它在我不知道的时候做了我不想让它做的事'。"

**具体表现**（这是本规范与传统 SaaS 规范最大的差异）：
- Agent 每一次**调用工具、读写数据、发送外部请求**，UI 必须同步呈现
- 任何 Agent 自主决策，都应展示 **Reasoning trace**（思考过程），可折叠
- 涉及**外部影响**的操作（发邮件、付款、改客户数据），必须 HITL 二次确认
- Streaming 过程中用户可以**随时中断**，中断按钮永远在视线内
- 每条 Agent 输出必须可追溯到 **citation / source**

详见 `07-agent-patterns.md`。

---

### Tenet 4 — Keyboard First（键盘优先）

> "专业用户一天按下 5000 次键，而他们按下鼠标 500 次。"

**具体表现**：
- 每个主要操作都有**键盘快捷键**，显示在 tooltip 和菜单项上
- **Command Palette（⌘K）** 是一等公民，不是藏在角落的功能
- `Tab` 顺序符合阅读顺序，`focus-visible` 视觉反馈醒目（不是浏览器默认的虚线）
- Table 行可用 `↑↓` 导航，`Enter` 打开详情，`Space` 选中，`Esc` 取消
- Modal 用 `Esc` 关闭，Confirm 用 `⌘Enter` 提交

---

### Tenet 5 — Editorial Warmth（编辑式温度）

> "冷冰冰的工具是 1990 年代的审美。2026 的专业软件要像一本 The New Yorker——严肃但可读、高密度但有呼吸。"

**具体表现**：
- 背景是**暖调 off-white**（`#FAFAF7`），不是纯白 `#FFFFFF`
- 标题用 **Fraunces** 衬线体点缀，body 用 **Geist** 保证可读
- 中文用 **微软雅黑**，因为它在高密度界面下字重层级最清晰
- 强调色是**深琥珀 Amber**，不是饱和蓝或紫——温暖 + 专业
- Radius 主用 **6–8px**，不走极简的 0px，也不走 UX Pilot 的 16–20px

---

## 三条 Anti-Principles（刻意避免）

| ❌ 避免 | ✅ 替代 |
|---|---|
| AI 产品标配的"紫色渐变 + 磨玻璃" | 暖米白背景 + 深琥珀 accent |
| 首屏大标题 + 滑动浏览的 landing 感 | 一屏见所有关键信息的 cockpit 感 |
| 笼统的"加载中..." | 显式进度 + 可取消 + 预计剩余时间 |

---

## Design Tenet 引用语法

AI 生成界面时，如果发生设计取舍，应在注释或说明中显式引用 Tenet：

```html
<!-- Tenet 2: Density — 这里使用 compact 行高 32px 承载更多记录 -->
<tr class="h-8">...</tr>
```

这不是装饰；这是让**下一个 AI 读到你的代码时**能理解设计意图、不做冲突修改。
